# 排体变形可视化 — 三维空间曲面网格展示 设计方案

> **版本**: v1.0  
> **日期**: 2026-05-12  
> **作者**: Senior Developer  
> **关联需求**: 软件修改文档(2026-05) — "排体变形可视化"功能模块  

---

## 一、需求概述

### 1.1 背景
当前系统已实现"应变"和"相对变形"两个展示模块。本需求新增"**排体变形可视化**"模块，用于**取代**原有的"相对变形"模块，将软体排整体的变形通过**立体可视的三维空间曲面网格**展示出来。

### 1.2 核心目标
- 将每个软体排（#19~#25）的变形数据，通过**六条3D曲线**（大小U的左右边+横边）生成一张**连续的网格曲面**。
- 在大U光缆的"宏观地形基准"之上，融合小U光缆的"高频细节"，生成能模拟**土工布张力膜特性**的平滑曲面。
- 前端以 Three.js / ECharts GL 等3D引擎渲染，区分大小U光缆颜色并在曲面表面高亮标出。

## 四、核心算法详细设计

### 4.1 数据准备阶段 (SkeletonDataPreparer)

#### 4.1.1 输入
- 已有的反演结果：`Map<String, List<PointDTO>>`（来自`calculateAllRowsDeformation`）
- 每个排体所有通道的形变数据

#### 4.1.2 步骤

**Step 1: 从已有反演数据中提取六条曲线**

从`PointDTO`列表中，按照通道号和坐标位置，将数据分组为六条曲线。识别规则：
- 根据通道号（CH前缀）和传感器坐标在软体排上的物理位置（Y坐标），判定属于大U左侧/右侧/横边 或 小U左侧/右侧/横边

```java
// 骨架曲线结构
public class SkeletonCurveDTO {
    private String rowBody;           // 排体号
    private String curveType;         // "LARGE_LEFT" | "LARGE_RIGHT" | "LARGE_BOTTOM" 
                                      // | "SMALL_LEFT" | "SMALL_RIGHT" | "SMALL_BOTTOM"
    private double[] zValues;         // 形变Z值序列
    private int requiredLength;       // 目标长度
    private double fixedY;            // 固定Y坐标（横边为null）
}
```

**Step 2: 传感器数据截取/补齐**

根据分组规则进行截取或补齐：

| 排体组 | 大U目标长度 | 小U目标长度 | 截取策略 |
|--------|:--------:|:--------:|----------|
| #19,20,24,25 | 195 | 90 | 取后195/90个点；不足则补齐 |
| #21,22,23 | 180 | 70 | 取后180/70个点；不足则补齐 |

补齐策略：
- 若实际传感器点数 N < 目标长度 L，在**首端**补齐 L-N 个点
- 补齐值 = 首端前3个有效值的线性外推平均

**Step 3: #19号排体大U右侧断裂特殊处理**

```java
// #19大U左侧有195个点（完好），右侧仅111个点（断裂）
int leftLen = 195, rightLen = 111;
int supplementCount = leftLen - rightLen; // = 84

// 取左侧前84个点补到右侧前端
double[] zLeftAll = getLargeULeft("19");
double[] zRightBroken = getLargeURight("19"); // 111个点

// 拼接：前84个用左侧数据，后111个用右侧断裂数据
double[] zRightFixed = new double[195];
System.arraycopy(zLeftAll, 0, zRightFixed, 0, 84);
System.arraycopy(zRightBroken, 0, zRightFixed, 84, 111);

// 关键：右侧前84个点仅用于插值，不参与可视化显示
// （前端将这些点标灰或不显示点标记）
```

### 4.2 大U光缆横边闭合 (BigUClosureService)

**目标**：确保大U左右两侧与横边在3D空间中形成闭合图形。

**算法步骤**：

```
输入: Z_LL[0..N-1], Z_LR[0..N-1], Z_LB[0..M-1]
输出: Z_LB_final[0..M-1] (闭合修正后的横边Z值)

1. 计算目标高差:
   Target_Diff = Z_LL[N-1] - Z_LR[N-1]

2. 计算横边自身高差:
   Current_Diff = Z_LB[M-1] - Z_LB[0]

3. 计算闭合误差:
   Error = Target_Diff - Current_Diff

4. 线性分配误差到横边:
   横边起点拼接在大U右侧尾端 (X=N-1, Y=30)
   for i = 0 to M-1:
       ratio = i / max(1, M-1)
       Z_LB_final[i] = Z_LB[i] + Error * ratio
```

坐标映射：
- 横边起始点：(X=N_large-1, Y=30)，结束点：(X=N_large-1, Y=10)
- 横边方向：从右→左（Y从30递减到10）

---

### 4.3 小U光缆主从基准融合 (SmallUFusionService)

**问题**：小U数据偏差起伏过大，直接用于曲面重构会产生大幅差异。

**解决思路**：以大U的"宏观地形"为基准线，在小U位置做线性插值得到理论基准，再将小U的高频细节叠加到基准上。

**算法步骤**：

```
输入: Z_LL[0..N_large-1], Z_LR[0..N_large-1], Z_SL[0..N_small-1], Z_SR[0..N_small-1]
输出: Z_SL_final[0..N_small-1], Z_SR_final[0..N_small-1]

Step 1: 截取大U的前N_small个点
  Z_LL_sub = Z_LL[0 : N_small-1]
  Z_LR_sub = Z_LR[0 : N_small-1]

Step 2: 通过横向距离比例线性插值，计算小U位置的"宏观基础高度"

  大U在Y=10和Y=30，小U在Y=15和Y=25
  
  小U左侧基准(Y=15, 距大U左10m的5/20=0.25处):
    Base_SL[i] = Z_LR_sub[i] + 0.75 * (Z_LL_sub[i] - Z_LR_sub[i])
    // 等价于: 0.75*Z_LL_sub[i] + 0.25*Z_LR_sub[i]
  
  小U右侧基准(Y=25, 距大U左10m的15/20=0.75处):
    Base_SR[i] = Z_LR_sub[i] + 0.25 * (Z_LL_sub[i] - Z_LR_sub[i])
    // 等价于: 0.25*Z_LL_sub[i] + 0.75*Z_LR_sub[i]

Step 3: 提取小U自身的高频细节（去趋势）
  窗口大小 = 15 (滑动平均)
  
  Detail_SL[i] = Z_SL[i] - movingAverage(Z_SL, 15)[i]
  Detail_SR[i] = Z_SR[i] - movingAverage(Z_SR, 15)[i]

Step 4: 主从融合（高频细节叠加到宏观基准线）
  Z_SL_final[i] = Base_SL[i] + Detail_SL[i]
  Z_SR_final[i] = Base_SR[i] + Detail_SR[i]
```

**小U横边闭合**：使用与"大U横边闭合"完全相同的算法，目标高差为 `Z_SL_final[N_small-1] - Z_SR_final[N_small-1]`。

### 4.4 幽灵节点构建 (GhostNodeService)

**目标**：防止3D渲染引擎在光缆外侧盲区（Y=0~10m 或 排尾区域）生成翘曲/乱飞的网格。

**构建4组幽灵节点**：

| 组名 | X范围 | Y | Z值来源 | 说明 |
|------|--------|:--:|---------|------|
| 右侧边缘 | 0 ~ N_large-1 | 40 | 复制 Z_LR | 完全复制大U右侧 |
| 左侧边缘 | 0 ~ N_large-1 | 0 | 复制 Z_LL | 完全复制大U左侧 |
| 尾部延伸区 | N_large ~ 200 | 0~40 | 复制 X=N_large-1 处大U横边的Z值 | 平直延伸 |
| 前端约束 | 0 | 0~40 | 线性插值 Z_LL[0]→Z_LR[0] | 前端横向拉平 |

```
详细逻辑：

组1: 右侧边缘 Ghost_Right
  for i = 0 to N_large-1:
      addPoint(x=i, y=40, z=Z_LR[i])

组2: 左侧边缘 Ghost_Left
  for i = 0 to N_large-1:
      addPoint(x=i, y=0, z=Z_LL[i])

组3: 尾部延伸 Ghost_Tail
  for x = N_large to 200:
      for y = 0 to 40 step 2:
          zRef = Z_LB 在X=N_large-1处的插值
          addPoint(x=x, y=y, z=zRef)

组4: 前端约束 Ghost_Front
  确保X=0处Y=0~40范围内有平滑过渡
  for y = 0 to 40 step 2:
      ratio = y / 40.0
      z = Z_LL[0] * (1 - ratio) + Z_LR[0] * ratio
      addPoint(x=0, y=y, z=z)
```

**注意**：幽灵节点标记为"约束点"，不参与最终曲面的可视化显示（透明或极淡色），仅用于约束插值边界。

### 4.5 曲面插值生成 (SurfaceInterpolationService)

#### 4.5.1 散点汇聚
将前面所有点（大U三边 + 小U三边 + 4组幽灵节点）汇总为一个空间离散点云集合：
```
PointCloud = { (x_i, y_i, z_i) | i = 1..K }
```

#### 4.5.2 插值算法选择

**推荐方案：自然邻点插值 (Natural Neighbor Interpolation)**

选择理由：
- 能生成**平滑曲面**，模拟土工布的张力膜特性
- 适合非结构化散点数据
- 在MATLAB中有成熟实现（`scatteredInterpolant` with `'natural'`），可参考移植
- 相比普通线性矩形插值，不会产生棱角分明的折面

**备选方案：薄板样条 (Thin Plate Spline, TPS)**
- 曲面更加光滑，但计算量较大
- 适合点数不太多的情况（本场景约2000~3000个散点）

#### 4.5.3 插值实现 (Java)

```java
/**
 * 自然邻点插值器
 * 基于Delaunay三角剖分 + Voronoi图权重计算
 */
public class NaturalNeighborInterpolator {
    
    // 使用 Apache Commons Math3 已有依赖
    // 构建 Delaunay 三角剖分
    
    /**
     * 在给定网格上执行自然邻点插值
     * @param points 散点列表 [(x,y,z), ...]
     * @param gridX  X轴网格点
     * @param gridY  Y轴网格点
     * @return Z值二维数组 [gridX.len][gridY.len]
     */
    public double[][] interpolate(List<double[]> points, 
                                   double[] gridX, double[] gridY) {
        // 1. 构建 Delaunay 三角剖分
        DelaunayTriangulation dt = buildDelaunay(points);
        
        // 2. 对每个网格点，找到其自然邻点
        double[][] zGrid = new double[gridX.length][gridY.length];
        for (int i = 0; i < gridX.length; i++) {
            for (int j = 0; j < gridY.length; j++) {
                double qx = gridX[i], qy = gridY[j];
                zGrid[i][j] = naturalNeighborInterpolate(dt, points, qx, qy);
            }
        }
        return zGrid;
    }
    
    private double naturalNeighborInterpolate(
            DelaunayTriangulation dt, List<double[]> points,
            double qx, double qy) {
        // 找到包含查询点的三角形
        // 计算每个自然邻点的Voronoi面积权重
        // 加权平均得到Z值
        double sumWeight = 0, sumZWeighted = 0;
        List<Integer> neighbors = findNaturalNeighbors(dt, qx, qy);
        for (int idx : neighbors) {
            double[] p = points.get(idx);
            double weight = computeVoronoiWeight(dt, points, idx, qx, qy);
            sumWeight += weight;
            sumZWeighted += weight * p[2];
        }
        return sumZWeighted / sumWeight;
    }
}
```

#### 阶段一：数据准备与骨架提取（预计2天）
1. 创建 `SkeletonDataPreparer` 工具类
   - 从已有反演结果中按通道号+物理位置分组提取六条曲线
   - 实现截取/补齐逻辑
   - 实现 #19 号排体断裂特殊处理
2. 创建 `SkeletonCurveDTO` 和 `SurfaceRequestDTO`
3. 编写单元测试验证各排体的六条曲线提取正确性

#### 阶段二：核心算法实现（预计4天）
1. 实现 `BigUClosureService` — 大U横边闭合
2. 实现 `SmallUFusionService` — 小U主从基准融合
3. 实现 `GhostNodeService` — 幽灵节点构建
4. 实现 `SurfaceInterpolationService`
   - 引入/实现自然邻点插值算法（推荐）
   - 或引入/实现薄板样条插值（备选）
   - 生成 201×81 的 Z 值网格
5. 编写各模块单元测试

#### 阶段三：服务编排与API接口（预计1天）
1. 创建 `SurfaceGenerationServiceImpl` — 编排上述各步骤
2. 创建 `SurfaceGenerationController` — REST API接口
3. 创建 `SurfaceMeshDTO` — 数据传输对象
4. 将 `/deformation/calculateSingle` 和 `/calculateAll` 标记为 `@Deprecated`

#### 阶段四：前端集成（预计3天）
1. 搭建 Three.js / ECharts GL 3D 场景
2. 实现曲面网格渲染
3. 实现大小U光缆高亮轨迹线
4. 颜色映射（根据Z值）
5. 交互控制（旋转/缩放/平移）
6. 响应式适配

#### 阶段五：联调测试与优化（预计2天）
1. 后端算法精度验证（与MATLAB结果对比）
2. 性能优化（大网格传输压缩、前端渲染帧率）
3. 边界情况测试（数据缺失、异常值）
4. #19断裂情况专项测试

### 6.2 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 自然邻点插值算法复杂度高 | 开发周期延长 | 优先实现TPS作为备选；TPS已有成熟Java库可用 |
| 插值算法精度与MATLAB不一致 | 曲面效果不达预期 | 用同一组数据对比，调试到偏差<5% |
| 201×81×7 个排体的JSON体积过大 | 前端加载慢 | 服务端gzip压缩；前端分页加载；降低网格精度(步长2m) |
| #19断裂段数据质量 | 曲面局部异常 | 加强幽灵节点在该区域的约束密度 |

### C. 插值算法参考

**MATLAB参考实现**:
```matlab
% 自然邻点插值
F = scatteredInterpolant(x, y, z, 'natural', 'none');
Z_grid = F(X_grid, Y_grid);

% 等效于在Java中实现基于Delaunay三角剖分的自然邻点插值
```

**伪代码**:
```
1. 对散点集构建 Delaunay 三角剖分
2. 对每个待插值网格点 (qx, qy):
   a. 找到包含该点的三角形
   b. 计算该点在该三角形中的重心坐标
   c. 插入该点后，重新计算局部 Voronoi 图
   d. 用面积权重加权平均得到Z值
3. 返回完整的Z值网格
```
